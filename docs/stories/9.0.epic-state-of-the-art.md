### Story 9.4 — openSMILE eGeMAPS + pyannote VAD (prosody deep) (P1, M)

**Duas integrações leves, mesma story.**

- **openSMILE eGeMAPS:**
  - Lib: `opensmile-python` (BSD, CPU-only)
  - Extrai 88 features acústicas padrão afetivo (Eyben et al. 2016)
  - Injeta em `voice_analyzer.py` + alimenta `tonality_analyzer.py`
  - Zero VRAM, zero custo
- **pyannote VAD:**
  - Lib: `pyannote.audio` (MIT) — requer HF token gratuito
  - Detecta pausas reais (não só silêncio do Whisper)
  - Classifica: micro-pausa (<0.4s), hesitação (0.4–1.2s), **pausa retórica (>1.2s)** ← Vinh highlighter
  - Novo campo no `voice_analyzer`: `pauses_retorical_count`, `pauses_strategic_ratio`
  - Alimenta `variety_analyzer` (pausas como variação)

**Esforço:** 18h
**Risco:** Baixo — libs estáveis, CPU ok

### Story 9.5 — py-feat FACS (facial 🟢 expandido) (P2, L)

- Lib: `py-feat` (MIT) — detector SVM-based rodável em CPU (ou PyTorch AU em GPU se couber)
- Substitui heurísticas smile/brow/EAR por **20 Action Units** do FACS (Ekman)
- Mapeamento AU → 6 emoções básicas (alegria, tristeza, raiva, surpresa, nojo, medo)
- Novo output em `facial_analyzer`: `emotion_distribution`, `au_variability`, `micro_expressions_count`
- `facial` continua 🟢 — agora com profundidade real pra alimentar `congruence_analyzer`
- **Resolve red flag Vinh:** "rosto = 40% da percepção emocional"

**Esforço:** 32h
**Risco:** Médio-alto — FACS é complexo, precisa validação com Gui

### Story 9.6 — Gemini Vision (gesto semântico) (P2, L)

**O único upgrade pago — ~$0.015/vídeo (Flash).**

- Provider: OpenRouter ou Google Cloud AI Studio
- Modelo: `gemini-2.5-flash` (custo) ou `gemini-2.5-pro` (qualidade)
- Input: 90 frames @ 1fps (vídeo 3min) + transcript sincronizado + prompt estruturado
- Output estruturado (JSON schema):
  - Por frame: `gesture_type`, `reinforces_message: bool`, `distracts: bool`
  - Global: `gesture_narrative_coherence_score` (0–100)
- Novo worker: `gesture_semantic_analyzer.py` — **augmentation, não scoring**
- Feature flag: `GESTURE_SEMANTIC_ENABLED=true/false` (default false até validar)
- Budget guard: `GESTURE_SEMANTIC_MAX_COST_PER_EVAL=0.10`

**Esforço:** 24h
**Risco:** Alto — dependência externa, custo variável, latência API

## Decisão: filosofia de calibração

Toda dimensão ML nova (9.3, 9.5) entra no pipeline como **augmentation** primeiro. Só vira scoring quando:
1. Gate 3 passou (correlação Pearson ≥0.75 com Gui)
2. Mentor (Gui) aprovou no protocolo blind Story 7.7
3. PR de promoção scoring/secundária tem review explícito

**Princípio Vinh:** "A aquisição de conhecimento traz satisfação. A aplicação traz realização." — integrar modelo é aquisição; validar com humano é aplicação.
