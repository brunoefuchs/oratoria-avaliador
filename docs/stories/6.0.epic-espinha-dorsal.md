# Epic 6: Espinha Dorsal — Melhor Avaliacao Possivel

## Status
Draft

## Contexto

Apos reuniao com o mentor (2026-04-10), decidiu-se focar em 5 features que formam a "melhor espinha dorsal possivel" para o APP. O objetivo e maximizar a qualidade da avaliacao e do relatorio ANTES de evoluir para features secundarias.

Spec arquitetural completa: `docs/architecture/epic-6-architectural-spec.md`

## Objetivo

Transformar o relatorio de "generico e isolado" para "personalizado, cruzado e hierarquizado". O orador deve sair com:
1. Um diagnostico CRUZADO (nao isolado) das suas metricas
2. Uma HIERARQUIA clara de problemas (nao lista plana)
3. Um SCORE DE IDENTIDADE baseado em como ele se posiciona na fala
4. Uma ANALISE DE ABERTURA com tecnicas de conexao detectadas
5. Tudo PERSONALIZADO pelo questionario reescrito pelo mentor

## Dependencias

- 6.Q deve ser implementada PRIMEIRO (todas as outras dependem dos novos campos)
- 6.1 e 6.2 podem rodar em paralelo (ambas sao prompt engineering)
- 6.8 depende de 6.Q (cruza desejo_transmitir com score de identidade)
- 6.10 nao tem dependencias tecnicas

## Metricas de Sucesso

- Reprocessar video de referencia (e4359918) e comparar relatorio antes/depois
- LLM gera insights cruzados (nao isolados) no texto
- Relatorio tem hierarquia clara (#1 CRITICO, #2 ALTO...)
- Score de Identidade aparece no dashboard
- Tecnicas de abertura detectadas (ou sugestoes se nao usou)
- Mentor valida qualidade do novo relatorio
