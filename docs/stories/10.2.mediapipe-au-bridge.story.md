## Acceptance Criteria

### AC1 — Fase 0: Spike (1 dia, standalone deliverable)

```
Given facial_analyzer.py
When MediaPipe FaceLandmarker é instanciado com output_face_blendshapes=True
Then blendshapes (52 coeficientes) ficam disponíveis no result
And spike script extract_blendshapes.py processa os 7 vídeos do bench
And output CSV salvo em docs/qa/blendshape-spike-2026-XX/{video}.csv
And cada CSV tem colunas: frame, timestamp, [52 blendshape names]
And README do spike resume: arquivos gerados, range valores observados, anomalies
```

### AC2 — Fase 1: Bridge module determinístico (2 dias)

```
Given módulo workers/_facs_blendshape_bridge.py
When map_blendshapes_to_aus(blendshape_dict) chamado
Then retorna AU dict com chaves AU1..AU45 (apenas as 22 mapeadas em 05-*)
And mapping é puramente determinístico (sem ML, sem fine-tuning)
And segue tabela exata de 05-mediapipe-au-bridge-spec.md
And thresholds por AU em config (pattern SMILE_INDEX_THRESHOLD existente)
And cross-check Duchenne genuíno: AU6 ativo SOMENTE se mouthSmileL/R simultâneo >= threshold
And derivada temporal aplicada pra detectar onset (não só intensidade absoluta)
```

### AC3 — Fase 2: Validação contra py-feat (1-2 dias)

```
Given py-feat 0.6.2 instalado em venv isolado Python 3.11
When rodar AMBOS (bridge novo + py-feat) nos 7 vídeos do bench
Then output frame-by-frame disponível pra ambos pra cada AU coberta
And calcular 4 métricas:
  1. Pearson por AU (frame-by-frame): bridge vs py-feat
  2. F1 binário (AU ativo/não, threshold por AU)
  3. IoU temporal (janelas 1s, pico em janela)
  4. Estabilidade: variância intra-vídeo (bridge ≤ py-feat × 1.5)
And resultado salvo em docs/qa/au-bridge-validation-2026-XX.md
```

### AC4 — Critério de promoção (gate Fase 3)

```
Given resultados Fase 2
When avaliar 10 AUs críticas (AU1, AU2, AU4, AU5, AU6, AU7, AU9, AU12, AU15, AU45)
Then critério de promoção GLOBAL:
  - Pearson médio ≥ 0.65 nas 10 AUs críticas
  - F1 binário médio ≥ 0.70 nas 10 AUs críticas
  - IoU temporal médio ≥ 0.55 nas 10 AUs críticas

Decisão automática:
  - Todas 10 passam → APROVADO global
  - 1-3 falham → APROVADO HYBRID (bridge para AUs aprovadas, py-feat ou heurística para falhas)
  - 4+ falham → REJEITADO global, voltar a OpenFace 3.0 license CMU comercial OU manter py-feat

Resultado documentado em docs/qa/au-bridge-promotion-2026-XX.md
```

### AC5 — Fase 3: Migração condicional via flag

```
Given critério promoção avaliado
When BLENDSHAPE_AU_BRIDGE_ENABLED=true
Then facial_analyzer + congruence_analyzer consomem AU dict do bridge
And quando flag false, comportamento idêntico ao pré-story (heurística atual)
And A/B test rodado por 1 sprint nos vídeos calibrados com Gui

Modo HYBRID (se aplicável):

  - py-feat ou heurística para AUs reprovadas
  - Dict resultado tem source: "bridge" | "pyfeat" | "heuristic" por AU
```

### Mapping table (10 AUs críticas + cross-checks)

| AU | Blendshape(s) | Threshold sugerido | Cross-check |
|---|---|---|---|
| AU1 | browInnerUp | 0.30 | — |
| AU2 | browOuterUpLeft + browOuterUpRight | 0.30 | — |
| AU4 | browDownLeft + browDownRight | 0.30 | — |
| AU5 | eyeWideLeft + eyeWideRight | 0.40 | — |
| AU6 | cheekSquintLeft + cheekSquintRight | 0.30 | **Duchenne**: AND mouthSmileL/R ≥0.40 |
| AU7 | eyeSquintLeft + eyeSquintRight | 0.30 | NÃO ativo se eyeBlink em mesmo frame |
| AU9 | noseSneerLeft + noseSneerRight | 0.20 | — |
| AU12 | mouthSmileLeft + mouthSmileRight | 0.40 | — |
| AU15 | mouthFrownLeft + mouthFrownRight | 0.30 | — |
| AU45 | eyeBlinkLeft + eyeBlinkRight | 0.50 | window-based (frame events) |

Tabela completa de 22 AUs em `05-mediapipe-au-bridge-spec.md` seção 2.

### Riscos e mitigações (de 05-*)

| Risco | Mitigação |
|---|---|
| Blendshape values são suaves (smooth coefficients) | Derivada temporal + smoothing window pra detectar onset |
| MediaPipe treinado em vídeo geral, não oratória PT-BR | Calibrar thresholds nos 7 vídeos do bench, não defaults universais |
| cheekSquint pode disparar sem AU6 real | Cross-check Duchenne com mouthSmile simultâneo |
| Vídeos handheld/mobile geram jitter | Reusar BROW_RAISE_DELTA_THRESHOLD = 0.005 (já calibrado mobile) |
| Latência | Blendshapes são output direto do FaceLandmarker — sem overhead |
| Cobertura FACS menor que py-feat | Documentar AUs faltantes; hybrid mode se análise depender |
