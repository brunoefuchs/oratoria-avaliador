# Story 9.4: openSMILE eGeMAPS + pyannote VAD

## Status
Ready for Review

## Size
M (~18h em 5 tasks)

## Business Value

Duas adições CPU-only (zero VRAM) que complementam voice/prosody analytics:

1. **openSMILE eGeMAPS** (88 features acústicas padrão afetivo — Eyben et al. 2016) — enriquece voice_analyzer com features academicamente validadas. Útil pra comparação com literatura + alimenta tonality_analyzer como features numéricas estáveis.

2. **pyannote VAD** (voice activity detection) — detecta pausas reais (não só silêncio do Whisper). Classifica em:
   - Micro-pausa (<0.4s) — natural de fala
   - Hesitação (0.4–1.2s) — sinal de nervosismo
   - **Pausa retórica (>1.2s)** — "highlighter verbal" do Vinh

Resolve o gap Vinh: "pausa = poder" — hoje só medimos silêncio bruto, não estratégico.

## Acceptance Criteria

### AC1 — openSMILE eGeMAPS integration

```
Given OPENSMILE_ENABLED=true (default false)
When voice_analyzer.extract_egemaps(audio_path) chamado
Then usa opensmile-python lib com feature_set="eGeMAPSv02"
And retorna 88 features numéricas em dict
And armazena em voice_analyzer output como campo "egemaps": {...}
And flag OFF preserva schema atual
```

### AC2 — pyannote VAD integration

```
Given PYANNOTE_VAD_ENABLED=true (default false)
When voice_analyzer.detect_pauses(audio_path) chamado
Then usa pyannote.audio pipeline "pyannote/voice-activity-detection"
And retorna lista de segments [{start, end, type}] onde type ∈ {micro, hesitation, retorical}
And classifica por duração: <0.4s=micro, 0.4-1.2s=hesitation, >1.2s=retorical
And total por tipo em voice_analyzer output
```

### AC3 — Alimentação de variety_analyzer

```
Given pause detection rodou
When variety_analyzer calcula monotonia
Then pausas retóricas entram como variação (pontos de alto signal)
And ratio pauses_retorical/total vira nova feature do variety score
And documenta trade-off: muitas retóricas = pausado demais
```

### AC4 — Feature flags independentes

```
Given OPENSMILE_ENABLED e PYANNOTE_VAD_ENABLED
When flags combinadas (ambas off, só smile, só vad, ambas on)
Then pipeline funciona em cada combinação
And logs indicam quais features foram computadas
```

### AC5 — Regressão + performance

```
Given flags OFF
When pipeline completo roda
Then 255+ regression PASS (baseline 9.2)
And performance bit-idêntico (CPU libs, lazy import)

Given flags ON
When pipeline roda em vídeo 3min
Then wall time overhead <= 20s total (CPU, paralelizável com GPU workers)
```

### AC6 — pyannote HF token requirement

```
Given pyannote.audio requer HF token gratuito
When PYANNOTE_VAD_ENABLED=true e HF_TOKEN ausente
Then log warning "pyannote requer HF_TOKEN — obter em huggingface.co/settings/tokens"
And cai em fallback silence-detection simples (librosa.effects.split)
And metric "pyannote_fallback_triggered" registrada
```
