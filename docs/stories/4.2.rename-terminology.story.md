# Story 4.2: Renomear Terminologia Tecnica para Linguagem de Oratoria

## Status
InProgress

## Executor Assignment
- executor: @dev
- quality_gate: @qa

## Story

**As a** orador avaliado,
**I want** que o relatorio use linguagem de oratoria que eu entendo,
**so that** consigo interpretar meus resultados sem precisar de conhecimento tecnico.

## Acceptance Criteria

1. Todas as 11+ substituicoes da tabela de escopo aplicadas no frontend E no backend (onde aplicavel)
2. Nenhum termo tecnico em ingles visivel no relatorio final (Pitch, Score, WPM, Fillers, Grounding, Default, Cluster)
3. Mesma string exata usada para cada conceito no dashboard, pagina de detalhe e texto gerado pelo LLM (ex: "Voz e Diccao" identico nos 3 locais)
4. Campos `fillers_per_minute` e `muletas_per_minute` unificados como "Vicios de Linguagem por Minuto" no frontend
