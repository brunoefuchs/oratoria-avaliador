## Story

**As a** colaborador que se auto-avaliou,
**I want** que a analise vocal inclua a textura emocional do som (nao so velocidade, volume, pitch, mas tambem tom valencia-ativacao-dominancia),
**so that** descubro se minha voz esta transmitindo a emocao certa — ja que minhas palavras podem dizer "estou entusiasmado" mas minha voz soar apatica ou tensa.

## Contexto

Handoff Vinh (P2 #11) identificou: o app cobre 4 das 5 Vocal Foundations (rate, volume, pitch, pause). Falta **Tonality** — textura emocional do som.

Frase chave do framework Vinh: *"Pitch variado nao prova variedade emocional. Voce pode ter pitch alto e mesmo assim soar monotonamente ansioso."*

Solucao: analise VAD (Valence/Arousal/Dominance) — taxonomia classica de emocao por audio.

Preferencia do usuario (Epic 7): **libs locais**. Candidatos:
- **opensmile** (extrator de features acusticas, maduro, gratuito)
- **pyAudioAnalysis** (mais alto nivel, inclui classificador pre-treinado)
- Custom + libs Python baicas (librosa + parselmouth — ja no stack)

## Acceptance Criteria

### Extracao de Features

1. **AC-1 Extracao de features acusticas**: Worker extrai features prosodicas do audio:
   - **Jitter** (variacao de pitch ciclo-a-ciclo)
   - **Shimmer** (variacao de amplitude ciclo-a-ciclo)
   - **HNR** (harmonics-to-noise ratio — rouquidao/tensao)
   - **Formants F1, F2** (timbre vocalico)
   - **Espectro de energia** (distribuicao espectral)
   - **Pitch contour** (ja existe em voice_analyzer.py — reutilizar)

2. **AC-2 Mapeamento para VAD**: Com base nas features, calcular 3 dimensoes:
   - **Valence** (-1 a +1): positivo = "animado/alegre", negativo = "triste/abatido"
   - **Arousal** (0 a 1): ativacao/energia — alto = "excitado", baixo = "calmo/apatico"
   - **Dominance** (0 a 1): controle — alto = "confiante/autoritario", baixo = "submisso/hesitante"

3. **AC-3 Classificacao em textura emocional**: Com base em VAD, categorizar o audio em texturas interpretaveis (nao emocoes explicitas, mas perfis):
   - `neutro` (V≈0, A moderado)
   - `entusiasmado` (V alto, A alto, D alto)
   - `confiante` (V moderado, A moderado, D alto)
   - `apatico` (V baixo, A baixo, D baixo)
   - `tenso` (V negativo, A alto, D moderado)
   - `hesitante` (V moderado, A baixo, D baixo)

4. **AC-4 Distribuicao temporal**: Calcular % do tempo em cada textura (janelas de 5s):
   - Exemplo: "Sua voz soou: 60% confiante, 25% neutra, 15% hesitante"

### Feedback para o Orador

5. **AC-5 Sugestao de acao**: Com base na distribuicao, gerar feedback:
   - Se >70% neutro: "Sua voz esta mantendo textura constante — aumentar variacao aumenta engajamento"
   - Se >30% tenso: "Pontos de tensao detectados — respirar antes de momentos criticos pode ajudar"
   - Se >50% apatico: "Energia baixa predominante — trabalhe generosidade de energia (ver Vocal Foundation 'Volume')"

### Output

6. **AC-6 Formato de output**: Campo `detailed_metrics.tonality`:
   ```python
   {
     "disponivel": bool,
     "score": int,  # 0-100 baseado em variacao saudavel de VAD
     "diagnostico": str,
     "vad_medio": {
       "valence": float,
       "arousal": float,
       "dominance": float
     },
     "vad_temporal": [
       {"start": float, "end": float, "v": float, "a": float, "d": float}
     ],
     "textura_distribuicao": {
       "neutro": float,
       "entusiasmado": float,
       "confiante": float,
       "apatico": float,
       "tenso": float,
       "hesitante": float
     },
     "textura_dominante": str,
     "feedback": str,
     "warnings": list[str]
   }
   ```

## Analise Tecnica

### Task 0 — Spike tecnico (2h) ⚠️ BLOQUEANTE

**Opcoes:**
1. **opensmile** (C++ + Python binding) — padrao industrial, ~100 features acusticas, oferece eGeMAPS (minimum feature set). Licenca: livre para uso academico/pessoal, paga para comercial. Verificar se MVP interno comporta.
2. **pyAudioAnalysis** — mais alto nivel, inclui classificador pre-treinado de emocoes. Python puro, MIT license.
3. **librosa + parselmouth (ja no stack)** — custom. Mais trabalho, mas sem nova dependencia.
4. **OpenSMILE via Docker container separado** — isolar licenca + overhead.

**Output do spike:**
- Decisao documentada em `docs/architecture/tonality-analyzer-decision.md`
- Sample de extracao VAD em 3 videos existentes
- Validacao qualitativa (voz calma vs excitada produz VAD diferentes)

### Mapping features → VAD

Literatura academica (Eyben et al., 2016 — eGeMAPS) mapeia:
- **Valence:** F0 mean, F0 stdev, jitter, HNR
- **Arousal:** loudness, F0 range, speech rate, spectral slope
- **Dominance:** F0 mean, intensity mean, spectral flux

Nao precisa treinar modelo — existem formulas validadas. Usar regressao linear simples ou heuristicas baseadas em thresholds tabelados.

### Output visual (AC-8)

Barra empilhada horizontal no drill-down de Voz/Diccao:

```
[confiante 60%][neutro 25%][hesitante 15%]
```

Ou grafico de pizza/radar se preferir. Decidir com @ux-design-expert se disponivel.

### Performance

- Features acusticas em audio 5min: ~15-30s em CPU
- Reutilizar audio ja extraido por video_processor.py (evitar re-extracao)
