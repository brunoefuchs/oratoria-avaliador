## Acceptance Criteria

### Extracao de Features

1. **AC-1 Extracao de features acusticas**: Worker extrai features prosodicas do audio:
   - **Jitter** (variacao de pitch ciclo-a-ciclo)
   - **Shimmer** (variacao de amplitude ciclo-a-ciclo)
   - **HNR** (harmonics-to-noise ratio — rouquidao/tensao)
   - **Formants F1, F2** (timbre vocalico)
   - **Espectro de energia** (distribuicao espectral)
   - **Pitch contour** (ja existe em voice_analyzer.py — reutilizar)

2. **AC-2 Mapeamento para VAD**: Com base nas features, calcular 3 dimensoes:
   - **Valence** (-1 a +1): positivo = "animado/alegre", negativo = "triste/abatido"
   - **Arousal** (0 a 1): ativacao/energia — alto = "excitado", baixo = "calmo/apatico"
   - **Dominance** (0 a 1): controle — alto = "confiante/autoritario", baixo = "submisso/hesitante"

3. **AC-3 Classificacao em textura emocional**: Com base em VAD, categorizar o audio em texturas interpretaveis (nao emocoes explicitas, mas perfis):
   - `neutro` (V≈0, A moderado)
   - `entusiasmado` (V alto, A alto, D alto)
   - `confiante` (V moderado, A moderado, D alto)
   - `apatico` (V baixo, A baixo, D baixo)
   - `tenso` (V negativo, A alto, D moderado)
   - `hesitante` (V moderado, A baixo, D baixo)

4. **AC-4 Distribuicao temporal**: Calcular % do tempo em cada textura (janelas de 5s):
   - Exemplo: "Sua voz soou: 60% confiante, 25% neutra, 15% hesitante"

### Feedback para o Orador

5. **AC-5 Sugestao de acao**: Com base na distribuicao, gerar feedback:
   - Se >70% neutro: "Sua voz esta mantendo textura constante — aumentar variacao aumenta engajamento"
   - Se >30% tenso: "Pontos de tensao detectados — respirar antes de momentos criticos pode ajudar"
   - Se >50% apatico: "Energia baixa predominante — trabalhe generosidade de energia (ver Vocal Foundation 'Volume')"

### Output

6. **AC-6 Formato de output**: Campo `detailed_metrics.tonality`:
   ```python
   {
     "disponivel": bool,
     "score": int,  # 0-100 baseado em variacao saudavel de VAD
     "diagnostico": str,
     "vad_medio": {
       "valence": float,
       "arousal": float,
       "dominance": float
     },
     "vad_temporal": [
       {"start": float, "end": float, "v": float, "a": float, "d": float}
     ],
     "textura_distribuicao": {
       "neutro": float,
       "entusiasmado": float,
       "confiante": float,
       "apatico": float,
       "tenso": float,
       "hesitante": float
     },
     "textura_dominante": str,
     "feedback": str,
     "warnings": list[str]
   }
   ```

### Mapping features → VAD

Literatura academica (Eyben et al., 2016 — eGeMAPS) mapeia:
- **Valence:** F0 mean, F0 stdev, jitter, HNR
- **Arousal:** loudness, F0 range, speech rate, spectral slope
- **Dominance:** F0 mean, intensity mean, spectral flux

Nao precisa treinar modelo — existem formulas validadas. Usar regressao linear simples ou heuristicas baseadas em thresholds tabelados.

**Mapeamento VAD heuristico:**
- Valence: jitter (negativo correlato), HNR (positivo correlato), pitch acima da media (positivo)
- Arousal: intensidade + range pitch + std intensidade
- Dominance: pitch baixo + intensidade estavel + HNR alto
- Calibrado por norms eGeMAPS (jitter ~1.2%, shimmer ~6%, HNR > 18dB)

**Decisoes tecnicas:**
- Window 5s: equilibra granularidade vs estabilidade estatistica de features
- Filtro de NaN em features (jitter/shimmer podem falhar em segmentos sem PointProcess valido) → fallback para neutral threshold
- Score formula: base 30 + diversidade (10/textura usada) - penalty default (>70%) - penalty negativa dominante
- 6 diagnosticos: tonalidade_rica/funcional/apatica/tensa/uniforme/limitada
