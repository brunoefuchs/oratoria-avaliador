## Acceptance Criteria

### AC1 — generate_report recebe AggregatedMetrics tipado (não dict)
```
Given aggregator retorna dict com overall_score: int | None + partial_aggregation
When generate_report eh chamado
Then aceita AggregatedMetrics (Pydantic) ou dict validado por TypeAdapter
And campos obrigatorios validados antes de chamar Gemini
```

### AC2 — Score None vira texto explicito no prompt (não "0/100")
```
Given overall_score=None ou dimension_scores tem faltantes
When prompt eh construido
Then prompt diz "Pontuacao: indisponivel (X de Y dimensoes falharam)" NAO "0/100"
And dimensoes faltantes aparecem explicitamente
And LLM NAO gera coaching em cima de scores ausentes
```

### AC3 — Identity check usa dimension_status, não score=50
```
Given identity agora retorna WorkerFailure quando falha (nao score=50)
When report avalia se inclui identity no prompt
Then usa isinstance(identity, WorkerSuccess) ou identity.dimension_status == "ok"
And NAO usa identity.get("score") != 50
And WorkerFailure de identity → prompt omite secao identidade
```

### AC4 — Retry com variacao
```
Given primeira tentativa falhou (JSONDecodeError)
When retry dispara
Then temperature varia entre tentativas (0.7 → 0.5 → 0.3)
And prompt adiciona instrucao "retorne APENAS JSON valido"
And log registra qual temp foi usada
```

### AC5 — Retrocompat removida
```
Given report novo gerado
When salvo no banco
Then retorna apenas schema novo (resumo, dimensoes) — nao mais summary + dimension_feedback
And consumers downstream (frontend) ja migrados pro schema novo
And llm_cost_usd real (nao 0.0 fake)
```

### AC6 — Feature flag dispatch
```
Given TRUTH_CONTRACT_ENABLED=true
When app.py chama report generator
Then chama generate_report() novo (Pydantic input)

Given flag=false
Then chama generate_report_legacy() preservado
```
