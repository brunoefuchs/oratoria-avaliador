## Acceptance Criteria

### Metricas Faciais Core

1. **AC-1 Smile frequency**: % do tempo em que o rosto apresenta sorriso (leve ou intenso).
   - Usar landmarks do Face Mesh para calcular "smile index" (distancia canto-boca vs canto-rosto)
   - Threshold calibrado: smile ativo quando smile_index > 0.4
   - Output: `smile_frequency_percent: float`

2. **AC-2 Smile variability**: Variacao no nivel do sorriso ao longo do video.
   - Calcular desvio padrao do smile_index em janelas de 5s
   - Diferencia sorriso "travado" (stdev baixo, 100% do tempo ativo) de sorriso "natural" (stdev moderado)
   - Output: `smile_variability: float` (0-1)

3. **AC-3 Brow movement**: Movimento das sobrancelhas como sinal de engajamento / surpresa.
   - Detectar eventos de elevacao rapida de sobrancelhas (landmarks 105, 334)
   - Contar eventos por minuto
   - Output: `brow_raises_per_minute: float`

4. **AC-4 Eye openness variation**: Variacao na abertura dos olhos (sinal de atencao vs fadiga).
   - Calcular eye aspect ratio (EAR) ao longo do video
   - Output: `eye_openness_stddev: float`

5. **AC-5 Diagnostico agregado**: Score 0-100 e diagnostico textual:
   - `rosto_estatico` (stdev baixo em todas metricas)
   - `expressao_monotona` (smile travado ou ausente)
   - `expressao_equilibrada` (variacao saudavel)
   - `expressao_exagerada` (movimento excessivo, pode distrair)

### Output

6. **AC-6 Formato de output**: Campo `detailed_metrics.facial`:
   ```python
   {
     "disponivel": bool,
     "score": int,
     "diagnostico": str,
     "smile_frequency_percent": float,
     "smile_variability": float,
     "brow_raises_per_minute": float,
     "eye_openness_stddev": float,
     "emocional_texture": {
       "neutro_percent": float,
       "positivo_percent": float,
       "tenso_percent": float
     },
     "feedback": str,
     "warnings": list[str]  # ex: "rosto nao detectado em 30% do video"
   }
   ```

### Congruence Feed

7. **AC-7 Integracao com Congruence Score**: Enviar dados faciais para `congruence_analyzer.py` (ja existe) para cruzamento com voz/corpo.
   - Exemplo: se voz soa triste mas rosto sorri 100% do tempo → flag de incongruencia
   - Nao alterar a logica do congruence_analyzer nesta story — apenas expor os dados em `detailed_metrics.facial` para ele consumir

### Integracao

8. **AC-8 Integrado ao pipeline**: Worker chamado em `ml-worker/app.py` no Step 3.5 (entre gestos e voz — depois que pose/face ja foram processados pelo video_processor).

9. **AC-9 Surface no relatorio**: Card "Expressao Facial" em `/report/[id]/page.tsx` (nivel de sub-dimensao, junto com Identidade/Abertura/Narrativa).
   - Mostra: smile frequency, diagnostico, feedback
   - Drill-down fica para enhancement futuro

### Fallback

## Analise Tecnica

### Pipeline extension

```python
# ml-worker/workers/facial_analyzer.py
import mediapipe as mp
import numpy as np
from typing import TypedDict

face_mesh = mp.solutions.face_mesh.FaceMesh(
    max_num_faces=1,
    refine_landmarks=True,
    min_detection_confidence=0.5,
)

SMILE_LANDMARK_IDS = {"mouth_left": 61, "mouth_right": 291, "jaw_left": 172, "jaw_right": 397}
BROW_LANDMARK_IDS = {"left_brow": 105, "right_brow": 334, "left_eye_upper": 159, "right_eye_upper": 386}

def analyze_facial(video_path: str, fps: int = 10) -> dict:
    """Extract facial features at reduced fps (10/s suficiente, economiza compute)."""
    frames = extract_frames(video_path, fps=fps)
    metrics_per_frame = []

    for frame in frames:
        results = face_mesh.process(frame)
        if not results.multi_face_landmarks:
            metrics_per_frame.append(None)
            continue

        landmarks = results.multi_face_landmarks[0].landmark
        metrics_per_frame.append({
            "smile_index": compute_smile_index(landmarks),
            "brow_elevation": compute_brow_elevation(landmarks),
            "eye_openness_left": compute_eye_aspect_ratio(landmarks, "left"),
            "eye_openness_right": compute_eye_aspect_ratio(landmarks, "right"),
        })

    return aggregate_metrics(metrics_per_frame, fps=fps)
```

### Smile index formula (indicativa — validar no spike)

```python
def compute_smile_index(landmarks) -> float:
    # Distancia horizontal canto-boca
    mouth_width = distance(landmarks[61], landmarks[291])
    # Distancia entre cantos do maxilar
    jaw_width = distance(landmarks[172], landmarks[397])
    # Smile index normalizado
    return mouth_width / jaw_width if jaw_width > 0 else 0.0
```

Threshold: smile_index > 0.4 = sorriso ativo (calibrar em videos reais).

### Performance

- 10 fps e suficiente (rosto humano nao muda dramaticamente em <100ms)
- Video 5min = 3000 frames a processar → ~30s em CPU moderna
- Considerar batch processing se ficar lento
