# Story 9.3: Wav2Vec2-Emotion (Tonality 🟡 → 🟢)

## Status
Ready for Review

## Size
L (~24h em 6 tasks)

## Business Value

Tonality hoje é heurística VAD via jitter/shimmer/HNR do Praat — classificação 🟡 Média confiança. Wav2Vec2 modelo treinado em ~140h de áudio emocionalmente anotado (MSP-Podcast dataset) retorna **Valence / Arousal / Dominance reais**, não derivados. Promove `tonality` de 🟡 pra 🟢 → candidato a voltar pra SCORING_DIMENSIONS após Gate 3 PASS.

**Modelo sugerido:** `facebook/wav2vec2-base-superb-er` (~400MB, cabe em margem 3.66GB) — mais conservador que `audeering/wav2vec2-large-robust-12-ft-emotion-msp-dim` (~1.3GB). Base = 6 emoções discretas; large = VAD contínuo. Decide no Pre-Flight.

**Benefício Vinh:** textura emocional vocal deixa de ser heurística especulativa e vira medição ML validada academicamente (Wav2Vec2 Hugging Face). Alimenta `congruence_analyzer` com sinal real.

## Acceptance Criteria

### AC1 — Wav2Vec2 emotion factory em MODEL_FACTORIES

```
Given workers/_model_loader.py
When import MODEL_FACTORIES
Then "wav2vec2_emotion" aponta pra factory real (não stub NotImplementedError)
And factory carrega modelo transformers + tokenizer
And primeira execução faz download automático (~400MB)
And vram_check.py com MODEL_ORCHESTRATOR_ENABLED reporta peak reais (não skipped_stub)
```

### AC2 — tonality_analyzer usa ML quando flag ON

```
Given TONALITY_ML_ENABLED=true
When tonality_analyzer.analyze_tonality chamado
Then usa ModelGPU("wav2vec2_emotion") pra inferir emoções
And output tem campos vad_ml: {valence, arousal, dominance} (floats 0-1)
And preserva campos heurísticos (vad_medio original) pra comparação
And adiciona emocao_distribuicao_ml: dict[str, float] (se base model)
```

### AC3 — Feature flag default FALSE

```
Given TONALITY_ML_ENABLED ausente ou false
When tonality_analyzer chamado
Then usa path heurístico puro (comportamento v0.7.0-alpha.2)
And output schema idêntico (sem vad_ml keys)
And zero impacto em VRAM/performance
```

### AC4 — Fallback se modelo indisponível

```
Given TONALITY_ML_ENABLED=true mas transformers lib ausente OU download fail
When tonality_analyzer tenta carregar
Then captura exception + log warning
And cai em path heurístico sem crash pipeline
And metric "tonality_ml_fallback_triggered" registrada
```

### AC5 — Regressão preservada

```
Given flag OFF
When suite tests ml-worker rodada
Then 255+ tests PASS
And comportamento tonality bit-idêntico Story 9.2
```

### AC6 — Novo teste ML path (mockado)

```
Given TONALITY_ML_ENABLED=true com modelo mockado
When analyze_tonality chamado
Then valida estrutura output (vad_ml keys presentes)
And telemetria VRAM logada
And unload pós-inferência (via ModelGPU context)
```
