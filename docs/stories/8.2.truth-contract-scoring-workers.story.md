# Story 8.2: Truth Contract — Scoring Workers Migration (Wave 1)

## Status
Ready

## Size
L (~12h em 8 tasks, execucao sequencial)

## Business Value

Story 8.1 estabeleceu a foundation (WorkerResult schema + migration 012 + repo + feature flag) e migrou variety como piloto. **Mas os 6 scoring workers que alimentam o aggregator ainda retornam dict com `score=0` fake em fallback.** Esses 6 workers compoem o `overall_score` — o numero que o usuario ve no relatorio. Se qualquer um mentir, o score final mente.

**Bug ativo critico:** `identity_analyzer` retorna `score=50` quando falha (nao 0 como os outros — mente ATIVAMENTE).

Apos esta story: 7 de 13 workers (variety + 6 scoring) retornam WorkerResult validado. Flag pode ser ativada pra scoring workers em producao.

**Impacto direto:**
- `overall_score` calculado apenas com dados verdadeiros
- Identity para de mentir score=50
- Frontend pode mostrar "dimensao indisponivel" pras 6 scoring dims
- Foundation pra 8.3 (augmentation workers) e 8.4 (aggregator rewrite)

## Dev Notes

### Pattern de Migracao (copiar do variety T3 da 8.1)

```python
# 1. Renomear funcao publica existente pra _compute_X_metrics()
# 2. Adicionar:

def analyze_X_legacy(...) -> dict:
    return _compute_X_metrics(...)

def analyze_X(...) -> WorkerResult:
    try:
        if _upstream_failed(input):
            return WorkerFailure(dimension="X", dimension_status="skipped", ...)
        result = _compute_X_metrics(...)
        return WorkerSuccess(dimension="X", score=result["score"], metrics=result["metrics"])
    except Exception as e:
        return WorkerFailure(dimension="X", dimension_status="crashed", failure_reason=f"{type(e).__name__}: {e}")
```

### Identity Fix Especifico
```python
# ANTES (bug):
except Exception:
    return {"score": 50, ...}  # MENTIRA

# DEPOIS (truth contract):
except Exception as e:
    return WorkerFailure(dimension="identity", dimension_status="failed",
                         failure_reason=f"identity analysis failed: {e}")
```
