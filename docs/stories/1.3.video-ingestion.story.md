## Story

**As a** system,
**I want** to receive an uploaded video and split it into separate audio and video streams,
**so that** each analysis engine can process its relevant stream independently.

## Acceptance Criteria

1. API leve (Render) dispara processamento no ML worker (HF Spaces) via HTTP POST com `evaluation_id` e `video_url`
2. ML worker baixa o vídeo do Supabase Storage
3. FFmpeg extrai áudio (WAV 16kHz mono) e mantém referência ao vídeo original
4. Arquivos processados ficam no filesystem temporário do ML worker
5. Status da avaliação atualizado para `processing` com substatus `splitting`
6. Em caso de erro no FFmpeg ou download, status atualizado para `error` com mensagem descritiva
7. Logging estruturado registra início, duração e resultado do split
8. Endpoint `GET /evaluations/{id}/status` retorna status atual com substatus

## Dev Notes

### Fluxo de Comunicação API ↔ ML Worker
[Source: architecture.md#architectural-patterns]

```
API (Render) → POST /process {evaluation_id, video_url, callback_url} → ML Worker (HF Spaces)
ML Worker → POST /evaluations/{id}/status-update {substatus} → API (para cada etapa)
ML Worker → POST /evaluations/{id}/callback {results} → API (no final)
```

**Autenticação:** Header `X-Callback-Secret` com valor de `CALLBACK_SECRET` env var. Ambos os serviços compartilham o mesmo secret.

### FFmpeg Commands
[Source: architecture.md#components]

```bash
# Extrair áudio WAV 16kHz mono (formato ideal para Whisper)
ffmpeg -i input.mp4 -ar 16000 -ac 1 -f wav output.wav

# Extrair frames a 2fps (para stories futuras - pose estimation)
ffmpeg -i input.mp4 -vf fps=2 frame_%04d.jpg
```

### ML Worker — Endpoint POST /process

```python
# Request body
{
    "evaluation_id": "uuid",
    "video_url": "evaluations/uuid/video.mp4",  # Path no Supabase Storage
    "callback_url": "https://oratoria-api.onrender.com/evaluations/{id}/callback"
}
```

### API — Endpoint GET /evaluations/{id}/status
[Source: architecture.md#api-specification]

```json
{
  "id": "550e8400...",
  "status": "processing",
  "substatus": "splitting",
  "progress": {
    "steps_completed": 0,
    "steps_total": 6,
    "current_step": "Separando áudio e vídeo..."
  }
}
```

**Steps:** splitting → analyzing_posture → analyzing_gesture → analyzing_voice → analyzing_fillers → generating_report

### Error Handling
[Source: architecture.md#error-handling-strategy]

- FFmpeg falha → status `error`, error_message descritiva
- Download do vídeo falha → status `error`, error_message "Falha ao baixar vídeo"
- ML worker indisponível → API retorna status `error` após timeout (30s)
