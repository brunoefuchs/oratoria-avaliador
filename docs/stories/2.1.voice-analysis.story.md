## Story

**As a** user,
**I want** my speech transcribed and analyzed for prosodic quality,
**so that** I can understand my speaking pace, vocal projection, and pitch variation.

## Acceptance Criteria

1. Whisper transcreve o áudio em PT-BR com timestamps por palavra
2. Transcrição armazenada na tabela `transcriptions` vinculada à avaliação
3. Parselmouth extrai: pitch médio (Hz), variação de pitch (semitones), intensidade média (dB), duração de silêncios
4. WPM calculado a partir dos timestamps da transcrição (total palavras / duração líquida)
5. Razão fala/silêncio calculada
6. Métricas armazenadas em JSON estruturado na tabela `analysis_results` com `dimension = 'voice'`
7. Status da avaliação atualizado com substatus `analyzing_voice`
8. Erro em Whisper ou Parselmouth não bloqueia pipeline — registra falha e continua

## Dev Notes

### DB Schema — transcriptions
[Source: architecture.md#database-schema]

```sql
CREATE TABLE transcriptions (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    evaluation_id UUID NOT NULL REFERENCES evaluations(id) ON DELETE CASCADE UNIQUE,
    full_text TEXT NOT NULL,
    words JSONB NOT NULL DEFAULT '[]',
    language TEXT NOT NULL DEFAULT 'pt-BR',
    model TEXT NOT NULL,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

**words JSONB format:**
```json
[
  {"word": "olá", "start": 0.0, "end": 0.4, "confidence": 0.98},
  {"word": "pessoal", "start": 0.5, "end": 1.1, "confidence": 0.95}
]
```

### DB Schema — analysis_results
[Source: architecture.md#database-schema]

```sql
CREATE TABLE analysis_results (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    evaluation_id UUID NOT NULL REFERENCES evaluations(id) ON DELETE CASCADE,
    dimension TEXT NOT NULL CHECK (dimension IN ('posture', 'gesture', 'voice', 'fillers')),
    score INTEGER CHECK (score >= 0 AND score <= 100),
    metrics JSONB NOT NULL DEFAULT '{}',
    confidence TEXT NOT NULL DEFAULT 'high' CHECK (confidence IN ('high', 'medium', 'low', 'failed')),
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    UNIQUE (evaluation_id, dimension)
);
```

### Whisper Usage Patterns
[Source: architecture.md#tech-stack, PRD Epic 2]

```python
import whisper

model = whisper.load_model(os.getenv("WHISPER_MODEL", "medium"))

result = model.transcribe(
    audio_path,
    language="pt",
    word_timestamps=True,
    verbose=False
)

# result["text"] -> full transcription string
# result["segments"] -> list of segments with start/end
# result["segments"][n]["words"] -> list of {word, start, end, probability}
```

- Modelo `medium` recomendado para PT-BR (bom equilíbrio qualidade/performance no HF Spaces 16GB)
- `word_timestamps=True` necessário para WPM e filler detection (Story 2.2)
- Audio input: WAV extraído por FFmpeg no passo de video split

### Parselmouth API — Pitch/Intensity Extraction
[Source: architecture.md#components]

```python
import parselmouth
from parselmouth.praat import call

snd = parselmouth.Sound(audio_path)

# Pitch (F0)
pitch = snd.to_pitch()
pitch_mean = call(pitch, "Get mean", 0, 0, "Hertz")
pitch_stdev = call(pitch, "Get standard deviation", 0, 0, "semitones")

# Intensity
intensity = snd.to_intensity()
intensity_mean = call(intensity, "Get mean", 0, 0, "energy")

# Silence detection
text_grid = call(snd, "To TextGrid (silences)", 100, 0.0, -25.0, 0.3, 0.1, "silent", "sounding")
n_intervals = call(text_grid, "Get number of intervals", 1)
# Iterate intervals to sum silent vs sounding durations
```

### Voice Score Calculation
Score 0-100 baseado em métricas combinadas:
- **WPM score:** Ideal 130-170 WPM para PT-BR. Penalidade proporcional ao desvio
- **Pitch variation:** Score mais alto = mais variação (evita monotonia). Ideal: stdev > 3 semitones
- **Intensity:** Score baseado em consistência (baixa variação = bom)
- **Speech/silence ratio:** Ideal 70-85% fala. Muito silêncio ou nenhum silêncio penaliza
