## Acceptance Criteria

### Core — CLI e Integracao com LLMs

1. **AC-1 CLI basico funcional**: Comando `pnpm convergence:run {video_id}` submete video aos 3 LLMs (Gemini 2.5 Flash, Claude Opus 4.6, GPT-5) e retorna JSON consolidado com scores normalizados.
   - Input: `video_id` (Supabase FK) ou `--video-path {path}` para videos locais
   - Output stdout: JSON formatado com `{llm, score_geral, scores_por_dimensao, pontos_fortes, pontos_fracos}` por LLM
   - Suporta `--verbose` para mostrar prompts enviados e raw responses
   - Trace: rastreia ao criterio Epic 7: "Convergencia media entre Gemini/Claude/GPT >= 85%"

2. **AC-2 Correlacao Pearson r entre notas gerais**: Harness calcula e exibe correlacao Pearson (r) entre os 3 scores gerais, com alvo >= 0.85.
   - Formula: `r = cov(X,Y) / (std(X) * std(Y))` aplicada pairwise (Gemini-Claude, Gemini-GPT, Claude-GPT) e media agregada
   - Output: tabela com r pairwise + media
   - Se < 0.85, marcar com `⚠️` e logar em `convergence_alerts`
   - Trace: Epic 7 "Correlacao entre notas gerais (alvo: r >= 0.85)"

3. **AC-3 Correlacao r por dimensao**: Mesma analise por cada uma das 6 dimensoes do relatorio (Voz/Dicção, Clareza, Presença, Gestos, Arquétipos, Congruência), alvo r >= 0.75.
   - Se alguma dimensao < 0.75, marcar `⚠️` naquela dimensao no output
   - Dashboard renderiza heatmap dimensao x LLM-pair
   - Trace: Epic 7 "Correlacao por dimensao (alvo: r >= 0.75)"

4. **AC-4 Concordancia top-3**: Calcula quantos dos top-3 pontos fortes e fracos coincidem entre os 3 LLMs, alvo >= 2 de 3 coincidem.
   - Algoritmo: string similarity via embedding (cosine >= 0.75 considera "mesmo ponto") OU substring match explicito
   - Output: contagem + exemplos dos pontos que divergem
   - Trace: Epic 7 "Concordancia em pontos fortes / fracos top-3 (alvo: 2 de 3 itens coincidem)"

### Persistencia e Dashboard

### Prompt unificado (source of truth)

Arquivo `tools/convergence/prompts/report-evaluation.ts` exporta prompt unico usado pelos 3 LLMs. Versionamento via SHA256 do conteudo permite rastrear: *"se mudou prompt, r pode ter mudado por isso — nao pelo analyzer"*.

```typescript
export const REPORT_EVAL_PROMPT_V1 = `
Voce e um avaliador tecnico de oratoria. Analise o video fornecido e retorne JSON com:
- score_geral: 0-100
- scores_por_dimensao: {voz, clareza, presenca, gestos, arquetipos, congruencia}
- pontos_fortes: top 3 (string[] de ate 3 items, concisos)
- pontos_fracos: top 3 (string[] de ate 3 items, concisos)

Criterios de avaliacao: [...]
`;

export const PROMPT_VERSION = sha256(REPORT_EVAL_PROMPT_V1).slice(0, 8);
```

### Rate limits e paralelismo

Cada LLM tem rate limit diferente. Configurar via env:
- `GEMINI_RPM=60` (default conservador)
- `CLAUDE_RPM=50`
- `GPT_RPM=30`

Usar `p-limit` ou `bottleneck` (libs ja comuns em Next.js) para throttle. No modo batch, processar LLMs em paralelo por video MAS respeitar limit individual de cada.

### Calculo de r (Pearson)

Pearson classico, normalizado para escala 0-1. Biblioteca recomendada: `simple-statistics` (leve, bem testada). Alternativa: implementar inline se preferir zero nova dependencia.

```typescript
import { sampleCorrelation } from 'simple-statistics';

function pairwiseR(scores: {[llm: string]: number[]}): {[pair: string]: number} {
  const pairs = [['gemini', 'claude'], ['gemini', 'gpt'], ['claude', 'gpt']];
  return Object.fromEntries(
    pairs.map(([a, b]) => [`${a}-${b}`, sampleCorrelation(scores[a], scores[b])])
  );
}
```

### Concordancia top-3 (similaridade semantica)

Para comparar se "pouca variacao de pitch" (Gemini) e "voz monotona" (Claude) sao o mesmo ponto, usar embeddings.

**Opcao A (recomendada — local, sem custo):** usar `@xenova/transformers` com modelo `Xenova/all-MiniLM-L6-v2` para embeddings + cosine similarity. Threshold 0.75 = mesmo conceito.

**Opcao B (mais preciso, mas custo LLM):** usar Voyage/OpenAI embedding API. Descartar por custo e por ironia (usar IA para medir convergencia de IAs → circular).

Usar Opcao A.

## Riscos e Mitigacoes

| Risco | Impacto | Mitigacao |
|-------|---------|-----------|
| Custo LLM explode em batch grande (ex: 100 videos x 3 LLMs) | Alto | AC-8 usa rate-limit e processa sequencial por video. Bruno controla quando rodar batch. Estimativa custo: ~$0.50 por video nos 3 LLMs (Gemini flash e barato, Claude Opus 4.6 medio, GPT-5 caro). Orcamento 10 videos = $5 |
| Prompt nao e o mesmo que `report_generator.py` usa internamente, gerando divergencia falsa | Alto | Task 3 faz audit do prompt atual + extrai criterios. Se divergem, criar PR pequeno em `report_generator.py` PRIMEIRO pra alinhar (pode virar micro-story 7.6.1) |
| Rate limit de LLM derrubar batch no meio | Medio | `bottleneck` com retry exponential + persistir progresso no Supabase (permite retomar de onde parou) |
| Embeddings locais (xenova) pesados no bundle do dashboard | Medio | Rodar embeddings APENAS no CLI (Node.js), nao no browser. Dashboard consome resultado ja calculado em `convergence_runs.top3_matches` (campo extra a considerar — pode ser Task 1.5) |
| Dashboard /dev/convergence vaza em producao | Baixo | Feature-flag `NEXT_PUBLIC_ENABLE_DEV_ROUTES` garante off em prod. Alternativa robusta: middleware que bloqueia `/dev/*` quando `NODE_ENV=production` |
| Top-3 concordancia da falsos positivos (embeddings dizem "mesmo" mas humano discorda) | Medio | Documentar threshold 0.75 como calibravel. Dashboard mostra exemplos divergentes pra review manual. Iterar threshold se necessario |
| Videos antigos no Supabase nao tem prompt_version — queries podem quebrar | Baixo | Schema `convergence_runs` e tabela nova, nao tem dado legado. Campo `prompt_version` e NOT NULL por design |
| Tres LLMs podem concordar entre si mas estar todos errados (correlacao alta, precisao baixa) | Alto — para o Epic | **Esta story nao resolve.** E exatamente o que 7.7 (ground truth humano) existe pra detectar. AC-10 menciona essa limitacao na documentacao |
