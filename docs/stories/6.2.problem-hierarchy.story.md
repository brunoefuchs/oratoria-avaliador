## Story

**As a** orador avaliado,
**I want** que meus problemas sejam rankeados por impacto real (#1 CRITICO, #2 ALTO...),
**so that** saio do relatorio sabendo EXATAMENTE qual e minha prioridade #1 (uma unica coisa pra focar).

## Acceptance Criteria

1. Nova funcao `_rank_problems(aggregated)` no report_generator.py
2. Usa tabela de pesos com pelo menos 11 metricas (conforme spec)
3. Calculo: priority = weight × severity (distance from ideal)
4. Retorna max 5-7 problemas ordenados
5. Cada problema tem: ranking, severidade (CRITICO/ALTO/MEDIO/BAIXO), label, valor atual, threshold ideal
6. Injetado no prompt ANTES dos insights cruzados
7. LLM instrudo: 80% do texto sobre #1 e #2, resto como observacao
8. Se nenhum problema (todos scores >= 70): retorna mensagem positiva
9. Orador sai com 1 unica prioridade: o problema #1
