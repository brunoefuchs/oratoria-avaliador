## Acceptance Criteria

### AC1 — Discourse arc analyzer module

```
Given novo módulo workers/discourse_arc_analyzer.py
When analyze_discourse_arc(transcript, context) chamado
Then usa Gemini 2.5 Flash em modo texto (input = transcript string, NÃO Vision/imagem)
  via SDK google-generativeai → genai.GenerativeModel("gemini-2.5-flash")
  com response_mime_type="application/json"
And prompt versionado em prompts/discourse_arc_v1.md (SHA registrado em metadata do output)
And temperature=0, top_p=0, seed=42 (determinismo)
And rubric explicita citando:
  - Toastmasters Pathways Content/Organization
  - TOEFL Topic Development
  - Matthew Dicks (specifics, dialogue)
  - Kindra Hall (sensory details)
And NÃO inventa critérios — todos rastreaveis a fonte
And output JSON estruturado:
  {
    "discourse_type": "lista" | "argumentação" | "narrativa" | "explicativo",
    "score": 0-100,
    "arc_label": str (categoria do arco — "incompleto" | "linear" | "arco_completo" | "circular_callback"),
    "tem_payoff": bool,
    "tipo_payoff": "insight" | "imagem" | "cta" | "lição" | None,
    "callback_abertura_fechamento": bool,
    "justificativa": str (cita trecho exato do transcript),
    "confidence": float 0-1
  }
```

## Dev Notes

### Por que Gemini Vision e não modelo local?
Memory `feedback_gemini_vision_on_demand.md`: "ML pago só em dims onde math é fraca, nunca default."

Discourse arc é **inerentemente narrativo/qualitativo**:
- Heurística de regex não captura arco completo (linear vs circular vs com callback).
- Modelo local seria custo enorme de calibração com n=7.
- Gemini com rubric explícita + temperature=0 entrega scoring rastreável.

Custo: ~$0.005-0.010 por eval. Em 1000 evals/mês = $5-10. Aceitável.

### Rubric explícita (esqueleto pra prompt versionado)

```
Você é um avaliador de discurso. Avalie o ARCO NARRATIVO MACRO do
texto fornecido seguindo rubric:

CRITÉRIOS (Toastmasters Pathways Content/Organization):
1. Tem início claro? (apresenta tema, contexto, hook)
2. Tem desenvolvimento? (progride com tensão/argumentação/exemplos)
3. Tem fechamento? (resolve, sintetiza, ou faz CTA)
4. Há callback abertura↔fechamento? (Matthew Dicks technique)

CRITÉRIOS (TOEFL Topic Development):
5. Coerência linguística (transições, conectivos)
6. Profundidade (não fica raso)

PAYOFF (Kindra Hall):
- Insight (revelação)
- Imagem (sensory anchor)
- CTA (action)
- Lição (moral)

Output JSON estruturado conforme schema. Cite trecho exato do
transcript em "justificativa".

Temperature: 0
Seed: 42
```

### Determinismo pra reprodutibilidade
- temperature=0 + seed=42 + top_p=0
- Prompt versionado em `prompts/discourse_arc_v1.md` (commit hash registrado em metadata do output)
- Qualquer mudança de prompt → bump versão → re-validar

### Validação cross-vídeo
- 7 vídeos não é muito mas é o bench oficial.
- Validar especificamente em GUI ITAJAI/PRIME (mentor padrão) antes de generalizar.
- Variance entre vídeos é teste de discriminação — se discourse_arc colapsa todos pra 75, sistema não está medindo nada útil.
