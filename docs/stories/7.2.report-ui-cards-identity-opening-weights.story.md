## Acceptance Criteria

### Identidade

1. **AC-1 Card Identidade visivel no relatorio principal**: Adicionar secao "Sua Identidade" como card de destaque em `/report/[id]` logo apos os 6 sub-scores das dimensoes principais.
   - Mostra: score 0-100 + diagnostico textual + 2-3 exemplos do transcript capturados pelo analyzer
   - Campos do backend (`detailed_metrics.identity`): score, diagnostico, exemplos_posicionamento, patterns_detectados

2. **AC-2 Drill-down Identidade**: Nova rota `/report/[id]/identity` com detalhamento:
   - Score + breakdown dos patterns detectados (ex: uso de "eu acredito", "na minha visao", "no meu ponto de vista")
   - Comparacao: usos primeira pessoa vs terceira pessoa
   - Sugestoes de melhoria se score baixo
   - Link "Voltar ao dashboard"

### Abertura

3. **AC-3 Card Abertura visivel no relatorio**: Adicionar secao "Sua Abertura" ANTES dos 6 sub-scores (posicao reforca que abertura e momento critico â€” conforme story 6.10 AC-9).
   - Mostra: score 0-100 + diagnostico + tecnicas detectadas (com icone verde) + tecnicas ausentes (icone cinza)
   - Campos do backend (`detailed_metrics.opening`): score, diagnostico, tecnicas_detectadas, tecnicas_ausentes, opening_text

4. **AC-4 Drill-down Abertura**: Nova rota `/report/[id]/opening` com:
   - Score + cada tecnica detectada (tipo, label, exemplo do transcript, qualidade)
   - Cada tecnica ausente com sugestao concreta
   - Trecho literal do opening (primeiros 20% do transcript)

### Transparencia de Pesos

5. **AC-5 Tooltip/Accordion no score geral**: Ao clicar no score geral (ex: 45/100) no topo do relatorio, abrir breakdown detalhado:
   - Lista de dimensoes com peso aplicado e contribuicao ao total
   - Exemplo: "Variedade Vocal (peso 25%): 14 Ă— 0.25 = 3.5 pontos"
   - Contexto detectado que influenciou pesos: "Avaliado para: vendas presencial"
   - Formula utilizada: "Pontuacao = ÎŁ(dimensao Ă— peso_contextual)"

6. **AC-6 Badge contextual fixo**: Abaixo do nome do orador / topo do relatorio, mostrar badge:
   - Formato: `<Badge>Avaliado para: {contexto}</Badge>`
   - Cor: neutra (cinza ou azul claro)
   - Tooltip ao hover explica: "Os pesos das dimensoes sao ajustados conforme o contexto declarado no questionario"

### Regressao Zero

7. **AC-7 Zero regressao**: Relatorios antigos que nao tem dados de identidade ou abertura (videos anteriores a 6.8/6.10) renderizam gracefully:
   - Se `detailed_metrics.identity` nao existe: card nao aparece (nao quebra UI)
   - Se `detailed_metrics.opening` nao existe: idem
   - Se `detailed_metrics.pesos_utilizados` nao existe: score geral nao mostra breakdown (tooltip vazio)

## Analise Tecnica

### Componente identity-card.tsx (esqueleto)

```tsx
type IdentityData = {
  score: number;
  diagnostico: string;
  exemplos_posicionamento: string[];
  patterns_detectados: { pattern: string; count: number }[];
};

export function IdentityCard({ data }: { data: IdentityData | null }) {
  if (!data) return null;
  return (
    <section className="card">
      <h2>Sua Identidade</h2>
      <ScoreBadge value={data.score} />
      <p>{data.diagnostico}</p>
      <ul>{data.exemplos_posicionamento.map(...)}</ul>
      <Link href={`/report/${id}/identity`}>Ver analise completa</Link>
    </section>
  );
}
```
