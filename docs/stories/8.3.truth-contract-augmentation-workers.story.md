## Business Value

Story 8.2 migrou os 6 scoring workers. Esta story completa a migracao dos **6 augmentation workers** — os que alimentam o report_generator com dados complementares (facial, tonality, opening, storytelling, temporal, congruence).

- Flag ON em producao — Truth Contract ativo
- Foundation pra 8.4 (Aggregator) e 8.5 (Report Generator) receberem dados tipados

## Dev Notes

### Diferenca vs Wave 1

Wave 1 (8.2) workers retornavam `{score, confidence, metrics}` (schema OG).
Wave 2 (8.3) workers retornam `{disponivel: bool, score, diagnostico, ...}` (schema intermediario).

A migracao pra WorkerResult e ligeiramente diferente:
```python
# Schema intermediario → WorkerResult
if disponivel:
    return WorkerSuccess(dimension="X", score=result["score"], metrics=result["metrics"])
else:
    return WorkerFailure(dimension="X", dimension_status="failed",
                         failure_reason=result.get("motivo", "analise indisponivel"))
```

### Post-8.3: O que muda em producao

Com flag ON por default:
- `analysis_results` grava `dimension_status` + `failure_reason` pra TODOS workers
- `_save_analysis()` dispatcha via isinstance (WorkerResult path)
- Legacy code (aggregator, report_generator) AINDA le via dict-like patterns — funciona porque `_save_analysis_legacy` nao eh chamado (flag ON), mas aggregator le do DB onde colunas legacy (score, confidence) continuam populadas por backwards compat no repo

**Proximo passo arquitetural: Story 8.4 (Aggregator) receber `list[WorkerResult]` direto em vez de ler do DB.**
