## Status
InReview

## Size
M (~8h em 5 tasks)

## Business Value

Com 8.1-8.3, todos 13 workers retornam WorkerResult. Mas o aggregator ainda recebe dicts via shim `_wr_to_agg()` (app.py L518-523) que **reintroduz score=0 fake** pra WorkerFailure. O `overall_score` continua mentindo.

## Dev Notes

### Signature Nova vs Antiga

```python
# ANTIGA (legacy — 6 dicts separados):
def aggregate_metrics_legacy(
    evaluation_id, posture_result, gesture_result, voice_result,
    filler_result, variety_result, archetype_result, video_metadata, ...
) -> dict

# NOVA (Truth Contract — dict de WorkerResult):
def aggregate_metrics(
    evaluation_id: str,
    results: dict[str, WorkerResult],
    video_metadata: dict,
    contexto: str | None = None,
    motivacao: list | None = None,
) -> dict  # retorna dict com overall_score: int | None + partial_aggregation: bool
```

### overall_score Logic

```python
# Extrair scores apenas de WorkerSuccess em scoring dims
for dim in SCORING_DIMENSIONS:
    result = results.get(dim)
    if isinstance(result, WorkerSuccess):
        dimension_scores[dim] = result.score
        detailed_metrics[dim] = result.metrics
    else:
        incomplete_dimensions.append(dim)

# Calcular overall
peso_total = sum(pesos[d] for d in dimension_scores if d in pesos)
if peso_total > 0:
    overall_score = round(sum(dimension_scores[d] * pesos[d] for d in dimension_scores if d in pesos) / peso_total)
else:
    overall_score = None  # ← VERDADE, nao 0

partial_aggregation = len(incomplete_dimensions) > 0
```
