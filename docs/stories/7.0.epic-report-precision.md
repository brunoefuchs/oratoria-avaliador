## Objetivo

Transformar a avaliacao de "medidor tecnico preciso dos canais isolados" em **avaliacao tecnica da musica completa** — cruzando canais (congruence ja pronta), capturando estrutura narrativa (storytelling), mapeando expressao facial (gap atual) e adicionando tonalidade emocional vocal (5a vocal foundation).

Alvo quantitativo: **convergencia >= 85% entre multiplas IAs (Gemini, Claude, GPT) avaliando o mesmo video**.

## Usuario e Motivacao

- **Estagio do produto:** MVP interno / piloto
- **Usuario primario:** colaborador da empresa se auto-avaliando (persona principal)
- **Usuario secundario:** dono/contratante com acesso a todas as avaliacoes (visao agregada)
- **Dor principal:** precisao tecnica da analise — "estamos no caminho certo, mas quero a analise mais precisa tecnicamente possivel"
- **Nao bloqueadores:** UX, onboarding, features sociais (cobertas em Epic 5)

## Restricoes Operacionais

1. **Libs locais > LLM** — priorizar MediaPipe Face Mesh, opensmile, pyAudioAnalysis (custo zero por video) antes de Gemini/Claude (custo por chamada)
2. **Libs de deteccao tem que rodar no container ml-worker** (Python 3.12, FastAPI, ja tem MediaPipe + Whisper + Parselmouth)
3. **Schema Supabase** — campos novos entram via migration, `detailed_metrics` aceita novos sub-campos sem breaking change
4. **Zero breaking change** em relatorios existentes — novas dimensoes se somam, nao substituem
5. **Time:** solo + Claude Code (AI-assisted, worktrees para paralelizar)
6. **Deadline:** nenhum — qualidade > velocidade

## Metricas de Sucesso

### Principal: convergencia multi-IA
Rodar harness que submete o mesmo video a Gemini 2.5 Flash, Claude Opus 4.6, GPT-5 e mede:
- **Correlacao entre notas gerais** (alvo: r >= 0.85)
- **Correlacao por dimensao** (alvo: r >= 0.75 por dimensao)
- **Concordancia em pontos fortes / fracos top-3** (alvo: 2 de 3 itens coincidem entre IAs)

### Secundarias
- Score geral do app vs score humano (mentor) em N=10 videos de calibracao: r >= 0.70
- Score de Congruence captura >= 80% das inconsistencias verbal-vocal-visual detectadas manualmente
- Cobertura do handoff Vinh: 12/12 itens enderecados (diretamente ou via backlog previo)

### Instrumentacao
- Novo dashboard `/dev/convergence` com resultados das rodadas multi-IA
- Log de acordo/desacordo entre IAs por video processado
- Aprovacao manual de baseline (10 videos) antes de shipar

## Specs das Stories Novas (handoff para @sm River)

### 7.6 — Report — Harness de Convergencia Multi-IA

**Prioridade:** CRITICAL (bloqueante para Wave 3)
**Tipo:** Infra/QA
**Esforco:** M (2-3 dias)
**Executor:** @dev
**Wave:** 1 (paralelo com 7.1)

**Objetivo:** Implementar harness CLI que submete o mesmo video a Gemini 2.5 Flash, Claude Opus 4.6 e GPT-5, mede convergencia e persiste em dashboard.

**Acceptance Criteria:**

- AC1: CLI `pnpm convergence:run {video_id}` submete video aos 3 LLMs (Gemini/Claude/GPT) e retorna JSON consolidado
- AC2: Calcula correlacao Pearson r entre notas gerais (alvo: r >= 0.85)
- AC3: Calcula correlacao r por dimensao (alvo: r >= 0.75 por dimensao)
- AC4: Calcula concordancia em top-3 pontos fortes e fracos (alvo: 2 de 3 itens coincidem)
- AC5: Persiste resultado em tabela Supabase `convergence_runs` com schema: `id, video_id, llm, score_geral, scores_por_dimensao, pontos_fortes_top3, pontos_fracos_top3, created_at`
- AC6: Dashboard `/dev/convergence` renderiza: tabela de runs + grafico de r por dimensao + diff visual entre LLMs
- AC7: Log de desacordo (quando r < 0.75) dispara entry em `convergence_alerts` para review manual
- AC8: API pode ser chamada em batch: `pnpm convergence:run --batch {video_ids.txt}` (para 7.7 usar)
- AC9: Zero impacto em producao: harness roda em path separado, nao interfere no pipeline regular
- AC10: Documentacao em `docs/dev/convergence-harness.md` com exemplos + como adicionar novo LLM

**Dependencias:** nenhuma (base stand-alone)
**Unblocks:** 7.3, 7.4, 7.5, 7.7

**Wave:** 2 (depois de 7.6)
